The core problem: Groupon app users open the app 1.5x per month and buy about 1x per year. 66% buy once and never return. We need a reason to keep them engaged beyond the moment they want a deal.

The solution: embed a Safari extension inside the Groupon app. Users save on every shopping site they visit. Groupon Bucks earned everywhere pull them back into the app. Every app install becomes a savings companion.

Free organic traffic with high savings intent. Today we capture $0.43 and lose the user. These solutions convert anonymous visitors into known users who feed into the app ecosystem above.

Solution A: Registration + Rewards

  • Convert anonymous to known via gift cards / rewards gating
  • Needs Groupon auth integration (Phase 1: 2-3 weeks)
  • $3.83/registration (BQ verified, Behdad)
  • 540K registrations/yr currently at ~1.8% rate
  • Revenue: $2.1-4.6M/yr

Solution B: Genie Desktop Extension + Rewards

  • Chrome extension for desktop users (39% of coupons traffic)
  • Already live: 981 installs, $6/user/yr validated
  • 34 MD rewards integration = 26-59x installs (8-18K/mo)
  • Proves the extension model that Safari inherits
  • Revenue: scale dependent

Solution C: Cashback

  • Evolved rewards for repeat purchases
  • % back as Groupon Bucks on every purchase
  • Builds on Bucks infrastructure (shared with Problem 1)
  • Needs scoping first
